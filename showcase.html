磁盘与文件管理

模拟 M×N 磁盘（64B×1024块），实现二级目录（MFD→UFD）、连续存储分配、空闲盘区链与位示图(Bitmap)双模式管理。支持文件 FCB/iNode 详情查看、FAT 链表视图、文件访问冲突演示，含内存缓冲页（LRU 置换）、文件保护机制与命令进程调度模式（文件操作生成命令进程，经调度后执行）。

页面置换策略

支持 LRU / FIFO / OPT / LFU / CLOCK 共 5 种页面置换算法，可视化帧状态与逐步/自动动画。新增算法对比模式，同一访问序列下柱状图对比缺页率与命中率，支持动画速度调节。

文件系统详解

二级目录 · 连续存储 · 位示图 + 空闲盘区链 · FAT 链表 · LRU 缓冲页置换

📂 二级目录结构

  • 主文件目录 MFD — 按用户划分
  • 用户文件目录 UFD — 每用户独立
  • FCB 包含：文件名、创建时间、权限、起始盘块号、文件长度
  • 支持目录查询、文件检索

💽 数据组织 — 连续存储

  • 文件数据存放在连续的磁盘块中
  • FCB 记录起始块号 + 占用块数
  • 支持顺序访问与直接访问
  • 读写通过内存缓冲页进行

🗂️ 空闲盘区管理 — 双模式

  • 空闲盘区链：按起始地址链接空闲区
  • 位示图(Bitmap)：64×16 网格可视化
  • FAT 链表视图：块号→块号→EOF
  • 分配时首次适应查找合适区域
  • 回收时自动合并相邻空闲区

📑 内存缓冲页 — LRU 置换

  • M×K 缓冲区（页大小 = 盘块大小）
  • 每页记录：所有者、访问时间、dirty 位
  • LRU 策略置换最久未访问页面
  • 脏页写回磁盘后再释放

模拟磁盘布局（M × N）

M = 64B / 盘块，N = 1024 盘块。前部存放目录与元信息，后部存放文件数据。

设计亮点 — 前端架构

纯 HTML 单文件实现的核心设计思路

单文件架构 — All-in-One HTML

所有 HTML 结构、CSS 样式、JavaScript 逻辑均内联在同一文件中，无需构建工具或服务器，双击即可在浏览器中运行。

CSS 变量主题系统 — Design Tokens

通过 :root 定义颜色变量（--accent、--green、--red 等），全局一致配色，修改一处即可调整整体主题。

异步动画引擎 — async/await + setTimeout

缓冲页加载、页面置换、进程调度等过程使用 async 函数配合延时实现逐步动画，代码可读性高。

Canvas 可视化 — 资源分配图 & 甘特图

死锁模块使用 Canvas 2D API 绘制进程-资源有向图，进程调度模块动态生成甘特图 SVG 可视化。

导览演示模式 — Demo Tour

一键启动自动导览，按顺序切换 6 个步骤（磁盘→页面置换→进程调度→银行家→同步），每步自动加载示例数据并执行核心操作，带进度条与步骤说明，适合答辩现场展示。

命令进程调度 — Command Process

磁盘面板可开启“进程模式”，文件操作不直接执行，而是生成命令进程节点（Ready→Running→Done），经调度后执行，实现文件系统与进程管理的耦合展示。

进程同步与互斥

多线程环境下的并发控制与资源管理

🔧 命令进程

磁盘面板可开启进程模式，文件操作（新建/删除/查看/修改）生成命令进程节点，状态从 Ready→Running→Done，经手动或自动调度后执行，展示文件系统与进程管理的耦合。

📡 进程通信

支持消息传递（FIFO 队列）、管道通信（有界缓冲区 + 读写端阻塞）、共享内存三种 IPC 方式，全面覆盖操作系统进程间通信机制。

🛡️ 同步互斥

生产者-消费者采用 mutex/empty/full 三信号量同步互斥，文件保护机制阻止并发访问冲突。

⏰ 进程调度

并发进程采用时间片轮转 RR 或优先级策略进行调度，支持动态优先级调整。
